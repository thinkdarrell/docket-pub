{# Category-landing item list partial. Returned standalone for HTMX cross-filter requests (HX-Request: true) so a filter swap re-fetches only the items section, not the full page (header + KPIs + timeline + filter chrome). The full-page ``category_landing.html`` includes this partial in place — non-HTMX requests render the full page with the item list inline, so deep links and bookmarks keep working unchanged. Required context (set by ``public.category_landing``): - items — list of agenda-item dicts for the page - next_offset — int or None; when set, drives load-more - cross_filters — list[str] of active cross-filter slugs (used by load-more to preserve filter state in the next-page URL) Wrapping element carries ``id="item-list"`` — that id is the HTMX ``hx-target`` for the cross-filter ``
