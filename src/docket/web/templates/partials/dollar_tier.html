{# Dollar Tier with WCAG Markup — spec §6.1, decisions #71 + #75. Triple-redundant tier signal: color (CSS class) + symbol ($/$$/$$$/$$$$) + screen-reader label (aria-label + sr-only). WCAG 2.1 AA compliant — tier perception works without color, without sight, and on monochrome printouts. Inputs (via implicit context — parent passes ``amount``): - ``amount`` — Decimal | None. When None or invalid, renders nothing. The partial calls the ``dollar_tier`` filter once at top to compute color / symbol / threshold-description, then templates them into the spec markup. The ``format_dollars`` filter handles US-formatted output (rendering ``$1.8M`` for ≥ $1M, ``$NNN,NNN`` otherwise). The parent ``aria-label`` is the canonical accessible name for assistive tech that doesn't traverse children. The visible text + ``sr-only`` span is the equivalent for tech that does. Some screen readers prioritize one; some prioritize the other — both must convey the same meaning. Note on possible double-announcement: a screen reader that respects ``aria-label`` AND traverses the visible children will announce the amount + tier name twice (once from the aria-label, once from the on-screen text + sr-only suffix). This is acceptable: it's a known trade-off in WCAG 2.1 AA implementations and the alternative — omitting either path — leaves users on the OTHER assistive-tech contract uncovered. Decision #75 explicitly chose double-coverage. CSS hooks emitted, no styles yet (later styling pass — same posture as E2 badges and E3 engagement strip): - ``.dollars`` — base - ``.dollars--green | --yellow | --orange | --red`` — tier modifiers - ``.sr-only`` — visually-hidden but screen-reader-readable (standard utility class; ``base.html`` should provide the styles at the design pass). Defensive against malformed input: the ``dollar_tier`` filter returns ``None`` for None / 0 / negative / bool / NaN / Infinity / non-numeric strings. The ``{% if tier_data %}`` guard short-circuits — no markup leaks for invalid amounts. Same posture as E4's source-anchor button rendering nothing when ``source_anchor`` is missing. #} {%- set tier_data = amount | dollar_tier -%} {%- if tier_data -%} {{ amount | format_dollars }} ({{ tier_data.symbol }}), {{ tier_data.color|title }} tier {%- endif -%}
