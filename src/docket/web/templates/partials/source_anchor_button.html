{# Source-anchor adaptive button — spec §6.4. Renders a "View Source" link that adapts to the captured anchor level: 1. PDF + bbox + page → PDF page link (browser honours ``#page=N`` natively, no JS). Today this branch produces the same href as the page-only branch — see TODO inside that branch about ``#page=N&viewrect=L,T,W,H`` when Stage 1 commits to PDF user-space coordinates for ``bbox``. 2. PDF + page → PDF page link. 3. PDF (bare) → bare PDF URL. 4. HTML + anchor → HTML page with ``#anchor`` fragment. 5. Video + timestamp → ``?t=NNN`` deep link, label includes the formatted timestamp via the ``format_timestamp`` filter. 6. Any ``anchor.url`` → bare "View Source" fallback. 7. ``data_quality == 'no_text_layer'`` → unavailable label, plus an admin-only link to the data-debt queue. 8. Otherwise → no markup. Every external link uses ``target="_blank"`` and ``rel="noopener noreferrer"`` (matches the convention from E1's ``_source_link_stub.html`` and E3's engagement strip). URL safety (defense-in-depth, two layers): * Scheme guard — only ``http://`` / ``https://`` survive. Stripped and lowercased for the *check*; the original-case URL goes into the ``href`` so we don't mangle case-sensitive paths or query values. This catches ``HTTPS://``, ``" https://..."`` (leading whitespace), and rejects ``javascript:``, ``data:``, and protocol-relative ``//host/path``. * Domain allowlist — static set of supported CMS / video hosts + dynamic set of municipality ``adapter_config->>'base_url'`` hosts loaded at app init. Even if a malformed row sneaks in with ``https://attacker.tld/...`` we won't render the link. See :file:`source_security.py` for the closed list and the leading-dot suffix-match logic. Both checks funnel through one ``is_source_url_safe`` Jinja global so a malicious anchor can't bypass the gate by claiming a specific ``type``. URL canonicalization before fragment/query concatenation: * Pre-existing ``#fragment`` is stripped before we append ``#page=N`` or the HTML anchor — otherwise the browser sees ``…#section1#page=7`` and ignores the second fragment. * The video branch picks ``?`` vs ``&`` based on whether the URL already contains ``?`` so e.g. a Granicus player URL with ``?clip_id=123`` gets ``&t=65`` appended, not a second ``?``. The bare-URL fallback branch passes ``_url`` through unmodified (fragment intact) since it's not appending anything. Defensive against malformed input: ``anchor`` is coerced via ``or {}`` so a missing/None ``source_anchor`` won't blow up. If the stored value is a non-mapping (string, list, etc.) ``.type`` / ``.url`` lookups return Undefined and silently fail every branch, landing on the no-render path — no traceback in production.
